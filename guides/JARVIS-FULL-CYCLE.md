# Jarvis v5.0 — полный цикл клиента

## Принцип

| Кто | Что видит |
|-----|-----------|
| **Клиент** | Только **Добряк** (Сергей Макаров) — мягкий, человечный стиль |
| **Менеджер в CRM** | **Jarvis** — аналитика, этап цикла, тексты под роли, запись в карточку |
| **CRM** | Заказы, оплаты, PDF, дашборд — без автосоздания заказов без «Приказ:» |

Клиентский портал (отдельный чат с Добряком) — в roadmap. Сейчас менеджер копирует тексты Jarvis в WhatsApp/Telegram.

---

## Этапы цикла

| Этап | Jarvis в CRM | Автоматизация сейчас |
|------|----------------|----------------------|
| **Новый лид** | Квалификация, вопросы | Сканирование полей из сообщений менеджера |
| **Консультация** | Анализ потерь, план | `investment_loss_kzt`, `recovery_*`, брокер → анкета |
| **Сбор документов** | Чеклист KYC | Роль AFM/AML, вкладка «Регулятор» |
| **Договор** | Черновик договора | Шаблоны obych / dla_posobia → PDF-редактор |
| **Оплата** | Напоминания | Заказ только «Приказ:» или вкладка «Оплаты»; давление — роли Банк/AFM |
| **Ведение кейса** | Дедлайны, документы | Command Center, задачи Jarvis |
| **Закрытие** | Итог, отзыв | Архив клиента |

Этап определяется автоматически в контексте CRM (`ЭТАП ЦИКЛА CRM: …`).

---

## Роли (переключатель в чате Jarvis на карточке клиента)

1. **Добряк** — по умолчанию  
2. **АРРФР**  
3. **AFM**  
4. **Банк**  
5. **Межд. регулятор**  
6. **Суд**  
7. **AML / комплаенс**

API: `GET /api/jarvis/roles`  
Чат: `POST …/jarvis/chat/stream` с `{ "message": "…", "role": "afm" }`

## Интерфейс CRM (текущий)

```
Карточка клиента
├── Обзор денег (Lichka vs Legal Consult)
├── Профиль (Alt+1) — анкета ~35 полей
├── Оплата (Alt+2) — заказы, client_paid, PDF
├── Jarvis (Alt+3)
│   ├── Переключатель роли
│   ├── Чат (stream + запись полей)
│   └── Анализ и задачи
└── Регулятор (Alt+4) — симулятор АРРФР
```

**Дашборд** — план дня, просрочки, «контакт был».

**Документы** — каталог, импорт наработок.

## Roadmap (следующие шаги)

1. **Клиентский чат** — виджет/API только «Добряк», Jarvis в фоне пишет в CRM.  
2. **Автопредложение роли** — «клиент молчит 5 дней → предложить AFM».

4. **Шаблоны по этапу** — кнопки «текст Добряка / банка / AFM».

## Обновление промпта на сервере

```bash
cd server && npm run db:refresh-jarvis-prompt
pm2 restart lichka-crm-api
```

Пересборка клиента после смены `VITE_*`:

```bash
cd client && VITE_API_URL=https://lichka.shop/crm VITE_BASE_PATH=/crm/ npm run build
```
