# Справочник администратора: требуемые документы кабинета

**Онлайн-версия с поиском:** админ-панель → **Инструменты → Справочник документов** (`/admin/document_guide.php`).

Документ синхронизирован с кодом: `IdentityDocuments::DOC_SLOTS`, `LEGAL_DOC_SLOTS`, `SourceOfFunds::templateSlotsForRegion()`. Цены услуг — ориентир из `platform_catalog_services` (могут меняться в БД).

---

## Как пользоваться

1. Клиент спрашивает «что за документ» — найдите **ключ** (`doc_passport_photo`, `kzt_income_certificate` …) в справочнике или по названию слота в кабинете.
2. Прочитайте блок **«Чем не заменять»** — это главный источник путаницы.
3. Скопируйте готовую фразу **«Ответ клиенту»** или адаптируйте.
4. **Обязательный** слот блокирует переход этапа, пока админ не поставит `approved`. **По запросу** — только если включён в профиле клиента или попросил compliance.

| Этап SDL | Страница клиента | Таблица БД |
|----------|------------------|------------|
| 1 Идентификация | `documents.php` | `identity_documents` |
| 2 Происхождение средств | `source_of_funds.php` | `sof_uploads` |
| 3 Юридический | `documents.php` (#legal-docs) | `identity_documents` (ключи `legal_*`) |

**Важно:** платные услуги каталога **не заменяют** загрузку файла в слот. Услуга помогает *получить* документ; результат всё равно загружается в кабинет.

## Похожие названия — шпаргалка

| Документ A | Документ B | Правило |
|------------|------------|---------|
| Паспорт — фото | Паспорт — прописка | Два разных обязательных файла этапа 1 |
| Прописка в паспорте | Свидетельство о регистрации | Прописка обязательна; свидетельство — опционально, по запросу |
| Справка по ИИН | Удостоверение личности | Предпочтительна гос. выписка по ИИН |
| Справка о доходах (KZT) | Справка с места работы | Первая — с суммами; вторая — о факте работы |
| Декларация в слоте SOF | Услуга «Подготовка декларации» | В слот — готовая принятая декларация |
| Выписка KZT/EU/US (этап 2) | Банковский пакет (этап 3) | SOF — движение средств; legal — письма банка под перевод |
| История кошелька crypto | Экспорт транзакций | Оба обязательны при зоне USDT |
| Пояснительная записка crypto | Услуга «Сводка по крипто» | Записку пишет клиент; сводка — платная опция |

## Этап 1 — идентификация (`documents.php`)

### `doc_passport_photo` — Паспорт, разворот с фотографией (обязательный)

- **Что:** читаемый разворот с фото, ФИО, датой рождения, номером паспорта РК.
- **Где взять:** фото/скан; PDF из eGov/«Цифровые документы».
- **Зачем:** KYC, сопоставление с анкетой, запуск проверок этапа 1.
- **Что даёт:** один из трёх обязательных документов до этапа 2.
- **Не путать:** со страницей прописки; с услугой `doc_passport_pack` (заверенные копии).
- **Услуга:** `doc_passport_pack` — ~25 000 ₸, 1–3 дня.
- **Ответ клиенту:** «Загрузите разворот с фотографией в Документы. Прописка — отдельным файлом.»

### `doc_passport_registration` — Паспорт, прописка (обязательный)

- **Что:** страница со штампом регистрации.
- **Где:** свой паспорт.
- **Зачем:** адрес, AML.
- **Не путать:** с `doc_residence_reg_cert`.
- **Ответ:** «Нужна страница с пропиской, не разворот с лицом.»

### `doc_iin_extract` — ИИН (обязательный)

- **Что:** справка/выписка по ИИН.
- **Где:** eGov, ЦОН, кабинет налогоплательщика.
- **Зачем:** идентификация в госреестрах.
- **Услуга:** `doc_iin_extract` — ~35 000 ₸, 2–7 дней.
- **Ответ:** «Нужна выписка по ИИН, не только номер на удостоверении.»

### `doc_residence_reg_cert` — Свидетельство о регистрации (опционально)

- **Что:** отдельный документ госоргана об адресе.
- **Где:** ЦОН, полиция, eGov.
- **Зачем:** если прописки в паспорте недостаточно.
- **Услуга:** ~28 000 ₸, 3–10 дней.
- **Ответ:** «Загружайте только по запросу менеджера.»

### `doc_birth_act_pack` — Акты ЗАГС (опционально)

- **Что:** рождение, брак, смена ФИО.
- **Где:** ЗАГС, eGov.
- **Зачем:** расхождение ФИО, семейные источники.
- **Услуга:** ~32 000 ₸, 5–14 дней.

## Этап 3 — юридический пакет (`documents.php#legal-docs`)

### `legal_notary_translation` (обязательный)

- **Что:** заверенный перевод для банка-получателя.
- **Где:** нотариус + бюро переводов.
- **Услуга:** `doc_translation_legal` — ~40 000 ₸, 2–5 дней.
- **Не путать:** с черновым переводом без печати.

### `legal_bank_accompaniment` (обязательный)

- **Что:** письма банка, подтверждения по сделке.
- **Где:** менеджер банка, интернет-банк «Справки».
- **Не путать:** с выпиской SOF этапа 2.

### `legal_beneficiary_details` (обязательный)

- **Что:** IBAN, SWIFT, имя получателя, назначение.
- **Где:** банк получателя, invoice.
- **Ответ:** «PDF из банка с полными реквизитами.»

### `legal_transfer_dossier` (опционально)

- **Что:** сводный transfer-pack.
- **Зачем:** сложные кейсы, по запросу специалиста.

## Этап 2 — происхождение средств (`source_of_funds.php`)

Активны только **зоны**, которые клиент выбрал в профиле активов (`sof_source_profile.assets_json`).

### Зона KZT (Казахстан)

| Ключ | Название | Обяз. | Кратко |
|------|----------|-------|--------|
| `kzt_income_certificate` | Справка о доходах | да | Суммы ЗП/дивидендов; услуга `doc_income_certificate` ~30k ₸ |
| `kzt_employment_letter` | Справка с работы | да | Должность и стаж, не заменяет доход |
| `kzt_tax_declaration` | Налоговая декларация | да | Принятая декларация; услуга `tax_declaration_prep` ~45k ₸ |
| `kzt_bank_statement` | Выписка KZT | нет* | 6–12 мес. по запросу; `doc_bank_statement` ~25k ₸ |
| `kzt_property_ownership` | Недвижимость | нет | При продаже жилья/земли |
| `kzt_investment_documents` | Брокер/ЦБ | нет | Торговля бумагами |
| `kzt_asset_sale_agreement` | Договор продажи актива | нет | Разовая выручка |

\*обязательность опциональных слотов может включить специалист.

## Типовые вопросы клиента — короткие ответы

**«Почему снова просите паспорт, я уже загружал?»**  
— Возможно отклонён (нечитаемо) или нужен **другой слот** (фото vs прописка). Проверьте статус в `identity_review.php`.

**«Можно одним файлом?»**  
— Нет для обязательной триады этапа 1: три отдельных ключа в БД.

**«Обязательно платить вам за справку?»**  
— Нет. Услуги опциональны. Самостоятельно: банк, eGov, работодатель → загрузка в слот.

**«Декларация есть, но в услугах просят оплату»**  
— Услуга = помощь в заполнении. Если декларация готова — только загрузка в `kzt_tax_declaration`.

**«Выписку уже давал на этапе 2, зачем банковский пакет?»**  
— Этап 2 = происхождение денег; этап 3 = документы **под перевод** (письма банка, реквизиты).

## Проверка админом

- Этап 1: `admin/identity_review.php?user_id=…`
- Этап 2: `admin/sof_review.php?user_id=…`
- Этап 3: legal-слоты в `identity_review.php` или `client.php` (блок документов)
- Карточка клиента: `admin/client.php?id=…`

При отклонении указывайте **конкретный ключ** и причину («перезагрузите `doc_passport_photo`: обрезан номер»), чтобы клиент не грузил не в тот слот.
